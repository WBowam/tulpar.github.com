git正常的工作流程

正常的工作流程

  1. 修改文件(在工作目录里)
    • 这时可以用git diff来查看工作目录内还没加到索引的变动
  2. 将它们更新的内容添加到索引中 git add file1 file2 file3.
    • 这时可以用git diff --cached查看已经被加到索引的变动
  3. 提交到仓库 git commit
    注: git commit -a包括了git add和git commit． 自动把所有内容被修改的文件(不包括新创建的文件)都添加到索引中，并且同时把它们提交

commit注释最好以一行短句子作为开头，来简要描述一下这次commit所作的修改(最好不要超过50个字符)；
然后空一行再把详细的注释写清楚。
这样就可以很方便的用工具把commit注释变成email通知，第一行作为标题，剩下的部分就作email的正文.
