数据库错误，如其他分支的数据库有改动，然后本地会报错column不存在，解决方法如下：

找到那个表所在的sql文件，在数据库里执行一遍即可，如报错在‘people_contact’表里缺字段，
找到最近的变动： grep -r 'people_contact' database/*.sql
结果如下：
`database/2014_11_10.camp.sql:ALTER TABLE `people_contact` ADD COLUMN `score` integer NOT NULL default 0; 是说10号增加了一个字段，database/2014_11_10.camp.sql内容如下:

 ALTER TABLE `people_contact` ADD COLUMN `score` integer NOT NULL default 0;
 ALTER TABLE `camp_camp` ADD COLUMN `price` integer NOT NULL default 0;
 ALTER TABLE `camp_campmember` ADD COLUMN `create_at` date NOT NULL;

将上面的几句在./manage.py dbshell里执行一下即可（快捷方法是直接source sql文件）

    'source_filenames':('css/listen_share.css',),
    'output_filename': 'css/listen_share.compress.css',
    },
  • 在template里引用
<head>                                                                                                
    {% compressed_css 'listen_share_css' %}                                                           
</head>
